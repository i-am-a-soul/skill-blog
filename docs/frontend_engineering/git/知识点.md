## `commit`的类型

- `feat`：新功能，会进`changelog`
- `fix`：修复，会进`changelog`
- `refactor`：重构（既不是新功能又不是修复），会进`changelog`
- `perf`：性能优化，会进`changelog`
- `docs`：文档或注释的改动
- `style`：格式如缩进、空格、换行等不影响代码运行的改动
- `test`：测试代码的改动
- `chore`：构建过程或辅助工具的改动
